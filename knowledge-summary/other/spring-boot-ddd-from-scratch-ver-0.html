Cấu hình pom.xml
File quan trọng nhất — khai báo dependency và plugin

Có 2 cách setup pom.xml. Cách thường dùng khi single module:

pom.xml — cách 1: dùng spring-boot-starter-parent (đơn giản nhất)
<project>
  <!-- Kế thừa từ Spring Boot parent → tự quản lý version -->
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.5.13</version>
  </parent>

  <groupId>com.example</groupId>
  <artifactId>my-project</artifactId>
  <version>1.0-SNAPSHOT</version>

  <properties>
    <java.version>21</java.version>
  </properties>

  <dependencies>
    <!-- Web = REST API + Tomcat embedded -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <!-- Không cần ghi version — parent đã quản lý -->
  </dependencies>

  <build>
    <plugins>
      <!-- Plugin này cho phép mvn spring-boot:run -->
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
    </plugins>
  </build>
</project>
pom.xml — cách 2: dùng BOM (multi-module, như tipjs series)
<project>
  <!-- Không dùng parent → import BOM thay thế -->
  <groupId>com.vetautet</groupId>
  <artifactId>vetautet.com</artifactId>
  <packaging>pom</packaging>

  <modules>
    <module>vetautet-start</module>
    <module>vetautet-controller</module>
    <module>vetautet-application</module>
    <module>vetautet-domain</module>
    <module>vetautet-infrastructure</module>
  </modules>

  <dependencyManagement>
    <dependencies>
      <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-dependencies</artifactId>
        <version>3.5.13</version>
        <type>pom</type>
        <scope>import</scope>
      </dependency>
    </dependencies>
  </dependencyManagement>
</project>
spring-boot-starter-parent Đơn giản hơn. Tự quản lý version. Không cần ghi version cho dependency. Dùng khi single module.
spring-boot-dependencies BOM Linh hoạt hơn. Dùng khi multi-module hoặc có parent POM riêng. Phải import vào dependencyManagement.
04
Main class — Application.java
Entry point — nơi Spring Boot khởi động
src/main/java/com/example/Application.java
package com.example;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
// = @Configuration + @EnableAutoConfiguration + @ComponentScan
// Tự động cấu hình Spring, tự scan toàn bộ package
public class Application {

    public static void main(String[] args) {
        // Khởi động Spring Context + Tomcat embedded
        SpringApplication.run(Application.class, args);
    }
}
@SpringBootApplication làm gì? Bật auto-configuration (tự detect và config bean), bật component scan (tự tìm @Service, @Controller...), bật @Configuration.
Đặt main class ở đâu? Đặt ở root package (com.example) — Spring scan từ đây xuống. Đặt sai chỗ → không tìm thấy Controller, Service.
05
Tạo Controller
Nhận HTTP request và trả về response
src/main/java/com/example/controller/HelloController.java
package com.example.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController           // = @Controller + @ResponseBody
@RequestMapping("/hello") // base path cho tất cả endpoint trong class này
public class HelloController {

    // GET /hello/hi → "Hello World!"
    @GetMapping("/hi")
    public String sayHi() {
        return "Hello World!\n";
    }

    // GET /hello/hi/{name} → "Hello, fongfox!"
    @GetMapping("/hi/{name}")
    public String sayHiTo(@PathVariable String name) {
        return "Hello, " + name + "!\n";
    }
}
  • ▸
    @RestController — tự động serialize return value thành JSON (hoặc String). Không cần @ResponseBody riêng.
  • ▸
    @RequestMapping("/hello") — base path. Tất cả endpoint trong class sẽ có prefix này.
  • ▸
    @GetMapping("/hi") — xử lý GET request tại /hello/hi. Tương tự: @PostMapping, @PutMapping, @DeleteMapping.
  • ▸
    @PathVariable — lấy giá trị từ URL path. Ví dụ /hello/hi/fongfox → name = "fongfox".
  • ○
    @RequestParam — lấy query param. Ví dụ /hello/hi?name=fongfox.
  • ○
    @RequestBody — lấy body từ POST request (JSON → object).

Cấu hình application.yml
File config chính của Spring Boot — port, DB, Redis...
src/main/resources/application.yml — config tối thiểu
server:
  port: 8080          # port chạy server (default 8080)
  tomcat:
    threads:
      max: 200         # max thread pool Tomcat

spring:
  application:
    name: my-app      # tên app (hiện trong log, Actuator)
application.yml — đầy đủ (khi thêm DB + Redis)
server:
  port: 1122

spring:
  application:
    name: vetautet.com

  datasource:           # MySQL config
    url: jdbc:mysql://localhost:3316/vetautet
    username: root
    password: root1234
    driver-class-name: com.mysql.cj.jdbc.Driver

  jpa:
    show-sql: true
    hibernate:
      ddl-auto: none    # none = không tự tạo/sửa table

  data:
    redis:              # Redis config
      host: 127.0.0.1
      port: 6319
yml vs properties yml (YAML) dùng indent để phân cấp → dễ đọc hơn. properties dùng dot notation. Spring hỗ trợ cả 2.
Lưu ý indent YAML Dùng 2 spaces (không dùng tab!). Sai indent → app fail start với lỗi khó hiểu.

Compile, package và khởi động ứng dụng

Dùng mvnw.cmd (Windows) hoặc ./mvnw (Mac/Linux):

Terminal — PowerShell (Windows)

.\mvnw.cmd clean install -DskipTests

# Bước 2: Run (single module)
.\mvnw.cmd spring-boot:run

# Bước 2: Run (multi-module — chỉ định module có main())

# Hoặc dùng IntelliJ: mở Application.java → click nút ▶ Run

    clean — xóa folder target/ trước khi build → tránh dùng class cũ.

  • ▸
    -DskipTests — bỏ qua test khi build để nhanh hơn.
  • ▸
    spring-boot:run — cần spring-boot-maven-plugin trong pom.xml của module có main().
  • ○
    -pl vetautet-start — chỉ run module cụ thể (dùng cho multi-module). Cần install trước.

09
Service Layer — Application + Domain
Business logic — nơi xử lý nghiệp vụ, interface + implement

Pattern chuẩn: luôn tạo Interface trước, Impl sau. Inject Interface — không inject class thẳng.

application/service/ticket/ITicketDetailAppService.java — Interface
package com.example.application.service.ticket;

import com.example.domain.model.entity.TicketDetail;

// Interface = bản hợp đồng, định nghĩa WHAT (làm gì)
// Prefix I: convention phổ biến trong Java/.NET enterprise DDD
public interface ITicketDetailAppService {

    TicketDetail getTicketDetailById(Long ticketId);
}
application/service/ticket/impl/TicketDetailAppServiceImpl.java — Implementation
package com.example.application.service.ticket.impl;

import com.example.application.service.ticket.ITicketDetailAppService;
import com.example.domain.model.entity.TicketDetail;
import com.example.domain.service.ITicketDetailDomainService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service   // Spring quản lý bean này, cho phép @Autowired
@Slf4j     // Lombok: tự inject logger, dùng log.info(), log.error()
public class TicketDetailAppServiceImpl implements ITicketDetailAppService {

    // Inject interface — không inject class thẳng!
    @Autowired
    private ITicketDetailDomainService ticketDetailDomainService;

    @Override
    public TicketDetail getTicketDetailById(Long ticketId) {
        log.info("Application layer — ticketId: {}", ticketId);
        return ticketDetailDomainService.getTicketDetailById(ticketId);
    }
}
domain/service/ITicketDetailDomainService.java + Impl
// Interface
public interface ITicketDetailDomainService {
    TicketDetail getTicketDetailById(Long ticketId);
}

// Implementation
@Service
@Slf4j
public class TicketDetailDomainServiceImpl implements ITicketDetailDomainService {

    @Autowired
    private ITicketDetailRepository ticketDetailRepository;

    @Override
    public TicketDetail getTicketDetailById(Long ticketId) {
        log.info("Domain layer — ticketId: {}", ticketId);
        return ticketDetailRepository.findById(ticketId).orElse(null);
    }
}
  • ▸
    @Service — đánh dấu đây là service bean. Spring quản lý lifecycle. Cho phép @Autowired.
  • ▸
    @Slf4j (Lombok) — tự sinh field log. Dùng log.info(), log.error(), log.debug().
  • ▸
    @Autowired — Spring tự inject bean phù hợp. Ưu tiên dùng Constructor Injection thay vì field injection.
  • ○
    Constructor Injection (chuẩn hơn) — thay @Autowired field bằng constructor có tham số → dễ test hơn.
Application vs Domain Service Application Service: điều phối use case, gọi cache + domain. Domain Service: chứa business rule, gọi Repository.
@Slf4j vs Logger thủ công @Slf4j (Lombok) = private static final Logger log = LoggerFactory.getLogger(...); — Lombok viết thay, ngắn hơn nhiều.

Domain Layer — Entity và Value Object
Model nghiệp vụ — Ticket, TicketDetail, các enum
domain/model/entity/TicketDetail.java — JPA Entity
package com.example.domain.model.entity;

import jakarta.persistence.*;
import lombok.*;
import lombok.experimental.Accessors;
import java.util.Date;

@Data              // Lombok: getter + setter + equals + hashCode + toString
@Accessors(chain = true)  // cho phép method chaining: obj.setName("x").setPrice(100)
@NoArgsConstructor // Lombok: constructor không tham số (JPA yêu cầu)
@AllArgsConstructor// Lombok: constructor đủ tham số
@Entity            // JPA: class này map với 1 table trong DB
@Table(name = "ticket_item")  // JPA: tên table trong DB
public class TicketDetail {

    @Id            // JPA: đây là Primary Key
    private Long id;

    private String name;
    private String description;
    private int stockInitial;
    private int stockAvailable;
    private Long priceOriginal;
    private Long priceFlash;
    private Date saleStartTime;
    private Date saleEndTime;
    private int status;
    private Long activityId;
    private Date updatedAt;
    private Date createdAt;
}
@Entity + @Table @Entity → JPA quản lý class này. @Table(name="ticket_item") → map với table có tên khác với class. Bỏ @Table → tên table = tên class.
Lombok annotations @Data = @Getter + @Setter + @EqualsAndHashCode + @ToString. Thêm Lombok dependency vào pom.xml + cài plugin trong IDE.
Domain không import Infrastructure Entity có @Entity (từ jakarta.persistence) nhưng KHÔNG import bất cứ thứ gì từ infra layer. Domain là trung tâm độc lập.
jakarta vs javax Spring Boot 3.x dùng jakarta.persistence (không phải javax.persistence). Nếu thấy lỗi import → kiểm tra đúng package.

Repository Layer — Truy cập Database
Interface trong Domain, Implementation trong Infrastructure

Repository pattern: Domain định nghĩa interface, Infrastructure implement. Domain không biết DB tồn tại.

domain/repository/ITicketDetailRepository.java — Interface (trong Domain)
package com.example.domain.repository;

import com.example.domain.model.entity.TicketDetail;
import java.util.Optional;

// Domain chỉ biết interface — không biết MySQL hay MongoDB
public interface ITicketDetailRepository {
    Optional<TicketDetail> findById(Long id);
}
infrastructure/persistence/mapper/TicketDetailJPAMapper.java — JPA (trong Infrastructure)
package com.example.infrastructure.persistence.mapper;

import com.example.domain.model.entity.TicketDetail;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;

// JpaRepository<Entity, IdType> — Spring Data tự generate SQL
// Có sẵn: findById, findAll, save, delete, count...
public interface TicketDetailJPAMapper extends JpaRepository<TicketDetail, Long> {

    // Spring Data tự generate: SELECT * FROM ticket_item WHERE id = ?
    Optional<TicketDetail> findById(Long id);

    // Custom query — Spring Data tự parse tên method
    // findByStatus → SELECT * FROM ticket_item WHERE status = ?
}
infrastructure/persistence/repository/TicketDetailInfrasRepositoryImpl.java
package com.example.infrastructure.persistence.repository;

import com.example.domain.model.entity.TicketDetail;
import com.example.domain.repository.ITicketDetailRepository;
import com.example.infrastructure.persistence.mapper.TicketDetailJPAMapper;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Repository;
import java.util.Optional;

@Repository  // dùng @Repository cho data access layer
@Slf4j
public class TicketDetailInfrasRepositoryImpl implements ITicketDetailRepository {

    @Autowired
    private TicketDetailJPAMapper ticketDetailJPAMapper;

    @Override
    public Optional<TicketDetail> findById(Long id) {
        log.info("Infrastructure layer — id: {}", id);
        return ticketDetailJPAMapper.findById(id);
    }
}
  • ▸
    JpaRepository<Entity, Id> — Spring Data tự generate CRUD SQL. Không cần viết SQL tay cho các query đơn giản.
  • ▸
    Optional<T> — tránh NullPointerException. Dùng .orElse(null) hoặc .orElseThrow() để lấy giá trị.
  • ▸
    @Repository — dùng cho data access. Spring wrap DB exception thành DataAccessException, dễ handle hơn.
  • ○
    @Query annotation — viết JPQL hoặc native SQL thủ công khi query phức tạp: @Query("SELECT t FROM TicketDetail t WHERE t.status = 1")

Infrastructure Layer — Config (Redis, Redisson, RestTemplate)
Đăng ký các bean kết nối với thế giới bên ngoài
infrastructure/config/RedisConfig.java
package com.example.infrastructure.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.*;

@Configuration  // Đây là class config — Spring đọc khi startup
public class RedisConfig {

    @Bean  // Khai báo bean — Spring quản lý, @Autowired được
    public RedisTemplate<Object, Object> redisTemplate(RedisConnectionFactory cf) {
        RedisTemplate<Object, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(cf);

        // Serialize key thành String, value thành JSON
        Jackson2JsonRedisSerializer serializer =
            new Jackson2JsonRedisSerializer(Object.class);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(serializer);
        template.afterPropertiesSet();
        return template;
    }
}
infrastructure/distributed/redisson/config/RedissonConfig.java
@Configuration
public class RedissonConfig {

    @Bean
    public RedissonClient redissonClient() {
        Config config = new Config();
        config.useSingleServer()
              .setAddress("redis://127.0.0.1:6319")
              .setConnectionPoolSize(50)
              .setDatabase(0);
        return Redisson.create(config);
    }
}
infrastructure/config/AppConfig.java — RestTemplate với timeout
@Configuration
public class AppConfig {

    @Bean
    public RestTemplate restTemplate() {
        // Cần timeout — không thì app bị đơ khi gọi external API chết
        HttpComponentsClientHttpRequestFactory factory =
            new HttpComponentsClientHttpRequestFactory();
        factory.setConnectTimeout(3000);  // 3s connect
        factory.setReadTimeout(3000);     // 3s read
        return new RestTemplate(factory);
    }
}
@Configuration + @Bean @Configuration: class này chứa bean definitions. @Bean: method này return một bean Spring quản lý. Dùng khi class không có @Service/@Repository.
RestTemplate cần timeout! Mặc định không có timeout → khi external API chết, app bị block vô tận → thread pool cạn → server sập. Luôn set timeout.

Kết nối toàn bộ — Full Call Chain
Controller → Application → Domain → Infrastructure → DB
controller/TicketDetailController.java — đầy đủ với response wrapper
@RestController
@RequestMapping("/ticket")
@Slf4j
public class TicketDetailController {

    @Autowired
    private ITicketDetailAppService ticketDetailAppService;

    // GET /ticket/1/detail/1
    @GetMapping("/{ticketId}/detail/{detailId}")
    public ResultMessage<TicketDetail> getTicketDetail(
            @PathVariable("ticketId") Long ticketId,
            @PathVariable("detailId") Long detailId) {
        log.info("GET ticket detail — ticketId:{}, detailId:{}", ticketId, detailId);
        return ResultUtil.data(ticketDetailAppService.getTicketDetailById(detailId));
    }
}
Full call chain khi gọi GET /ticket/1/detail/1
 
1. TicketDetailController ← nhận request, gọi App Service
   ↓
2. TicketDetailAppServiceImpl ← điều phối: gọi cache hoặc domain
   ↓
3. TicketDetailCacheService ← check Redis → hit? return. miss? lock → query
   ↓ (cache miss)
4. TicketDetailDomainServiceImpl ← business rule, gọi Repository
   ↓
5. TicketDetailInfrasRepositoryImpl ← implement ITicketDetailRepository
   ↓
6. TicketDetailJPAMapper ← Spring Data JPA → SQL → MySQL
   ↓
7. MySQL DB ← SELECT * FROM ticket_item WHERE id = 1
Luồng response ngược lại MySQL → JPA → Infra → Domain → Cache (set Redis) → Application → Controller → Client. Mỗi layer wrap thêm logic của mình.
Log ở mỗi layer Dùng @Slf4j + log.info() ở từng layer. Khi debug sẽ thấy rõ request đi đến đâu và bị lỗi ở layer nào.

Troubleshooting — lỗi hay gặp khi mới bắt đầu
  • ✗
    No plugin found for prefix 'spring-boot' — Thiếu spring-boot-maven-plugin trong pom.xml của module có main().
  • ✗
