Whiteboard mô tả Cache Stampede và hai giải pháp Cache-Aside + Distributed Lock

Whiteboard: Cache stampede & solution — bài toán mở bán vé tàu Tết 12:00

1. Vấn đề — DB không chịu nổi tải đọc

Ghi chú:

  • 50,000 người F5 lúc 12:00
  • Cache Redis hết TTL
  • N threads cùng miss
  • N threads cùng query DB

→ Cache Stampede!
→ DB sập!

Gặp ở mọi domain:
flash sale, vé phim,
tỷ giá ngân hàng...

2. Giải pháp

A. Cache-Aside (Normal)

Request
↓
Kiểm tra Redis
miss ↓
Query DB
↓
Set Redis
hit →
return
Vấn đề: N threads cùng miss!
→ N queries tới DB = Stampede!

B. + Distributed Lock (VIP)

Request
↓
tryLock (Redisson)
win ↓
Check Redis lần 2
↓
Query DB
↓
Set Redis + unlock()
lose →
return null
(chờ tạm)
finally { unlock() } — BẮT BUỘC!
Dù thành công hay exception cũng phải gọi
